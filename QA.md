## Coordinates

- **Public URL**: `https://yay-tsa.com` (Subsonic at `/rest/*`, API at `/api/*`).
- **Repo topology**: `origin` = Forgejo (`git.nikolay-eremeev.com/nikolay-e/yay-tsa`), GitHub is the push-mirror (the project CLAUDE.md line claiming GitHub origin is wrong). Push fixes to Forgejo; `gh` still works for issues/PRs/`gh run` (all on the GitHub mirror). Dependabot PRs open + merge on GitHub and sync back — a local `main` based on a pre-merge SHA shows "ahead N, behind M": rebase onto `origin/main` before pushing. `gh pr merge` on a dependabot PR is correct here.
- **Namespaces**: `yay-tsa` (current prod — backend, frontend, audio-separator, feature-extractor pods all live here) plus legacy `yay-tsa-v2-production` references in some charts/scripts. CNPG cluster in `shared-database`; replicas reject writes (`read-only transaction`) — reads are fine on any pod.
- **OpenAPI**: springdoc default `/api/v3/api-docs` (NOT Jellyfin's legacy `/api-docs` — the legacy path 401s and kills `tools/post-deploy-qa.sh` at the curl step). schemathesis `--url https://yay-tsa.com/api`.
- **SonarCloud key**: `nikolay-e_yay-tsa` (org `nikolay-e`). The `qualitygates`/`issues` APIs 404 on any other key — verify with `api/components/search?organization=nikolay-e&q=yay-tsa`.
- **QA account**: username `test`; password in Keychain entry `yay-tsa-qa-password`. The SAME value feeds the in-cluster `autoqa-crawler-login` secret (namespace `argo-workflows`) — if either is rotated, rotate BOTH and re-hash the `test` user's DB row (BCrypt cost 13). Subsonic `/rest/*` uses query-param auth (u/p/t), so a Bearer token 401s there — expected. NEVER use the `master` account (owner/admin; browser may autofill it) — admin login is forbidden during `/qa`.
- **DB schemas**: per-context `core_v2_*` (`auth`, `library`, `playback`, `adaptive`, `preferences`, `playlists`, `ml`, `karaoke`, `shared`) in `yaytsa_production`. `public` holds only extensions (pgvector, pg_trgm, citext, uuid-ossp).
- **Backend is `replicas: 1` — HARD INVARIANT** (per-JVM state). Every rollout has an unavoidable ~40s window (Flyway + Tomcat) where Cloudflare serves its own 502 HTML.

## Chart deploy path is git-direct

The ArgoCD apps consume charts **directly from the Forgejo git repo** (`repoURL: …/yay-tsa.git`, `path: charts/<chart>`, `targetRevision: main`) with values from gitops. The gh-pages Helm repository is legacy for external users only — a stale `index.yaml` does NOT mean undeployed chart changes. Verify a chart change is live by diffing the rendered live object (`helm template` vs `kubectl get`), not the published chart version. When a live configmap/deployment lacks a key the current template renders, suspect a stale/unsynced chart before suspecting the template.

- **ksops secret edits can show a false ArgoCD `Synced`**: after `sops set` adds a key, `secrets-yay-tsa` may report Synced while the live Secret lacks it. Force: `kubectl patch application secrets-yay-tsa -n argocd --type merge -p '{"operation":{"sync":{"revision":"HEAD","syncStrategy":{"apply":{"force":true}}}}}'`. A pod referencing the missing key fails container-create (`couldn't find key … in Secret`) and recovers once it lands.

## Internal autoqa (Argo Workflows)

Post-deploy QA is an in-cluster Argo Workflow (`yay-tsa-autoqa-*`, namespace `argo-workflows`), NOT a GitHub Action — no autoqa pin in `.github/workflows/`, no `gh run` for it.

- **Reading a run**: `kubectl get workflows -n argo-workflows` — the newest run whose `spec.arguments.parameters[commit-sha]` == deployed SHA is authoritative. `kubectl logs -n argo-workflows <pod> -c main` (failed pods survive; `podGC: OnWorkflowSuccess`). `.status.nodes` is a MAP not an array — array-form jsonpath errors.
- **GC'd runs**: the workflow archives `main.log` to Minio (`archiveLogs: true`); once GC'd, list the bucket by LastModified (`aws s3api list-objects-v2 --bucket argo-workflows-artifacts` via a minio port-forward; creds live in secret `argo-artifacts-minio`). Keys are `<workflow>/<pod>/main.log`; the file may or may not be gzipped. See the `/qa` skill for the generic recipe.
- **autoqa version = the workflow image ref**, a **hand-pinned full SHA** in `gitops:kubernetes/ci/ci-platform/workflow-templates/autoqa.yaml` (ci-platform has NO Image Updater annotations). After pushing to `nikolay-e/autoqa`, bump the pin in gitops (verify the tag exists first); workflows pick the template at submit time — in-flight runs keep the old image.
- **The gate** prints `## AutoQA — final gate` with a per-tool table + `### Findings`. It reconciles to informational: schemathesis FAILURES blocks whose statuses are all CF-edge 502/504/52x with a Cloudflare page body, and `positive_data_acceptance` blocks answered by structured problem+json 4xx (autoqa#29/#30); stateful-phase sections are re-split per test case (autoqa#44). Origin 5xx and non-problem bodies still block. A small residual `N` usually hides ONE real conformance bug among reconciled noise — read every `FAILURES` block, not just the first.

### Transient classes (verify before touching app code)

- **Rollout-window CF 502s**: the sensor fires on the Forgejo push webhook immediately, and the single-replica backend restart guarantees a 502 window. schemathesis "Undocumented 502", ZAP "report missing" (OpenAPI fetch got 502), monkey "failed request" bursts — all the same class. Confirm: 502 body is Cloudflare HTML (`server: cloudflare`), backend pod has 0 restarts, flagged endpoints answer documented statuses on replay. The decisive clean run is one submitted with no rollout in flight (image tags == target SHA, pods stable ≥1min).
- **Load-induced CF 502/520**: a bulk karaoke requeue keeps the GPU sidecar at ~1 CPU for hours; QA runs in that window catch edge 5xxs that don't reproduce on replay. Requeue in bulk only after confirming the batch runs off-scheduler; expect the first QA run after to be noisy.
- **JVM warm-up read-timeouts**: a lone schemathesis Network Error (10s read-timeout) within ~60s of the backend pod coming up replays in ~0.1s — cold JIT/Hikari, not an app bug (upstream retry tracked as autoqa#35). Replay the exact curl from the ERRORS block against the settled backend first.
- **Preempted zombies**: rapid pushes leave superseded runs alive at `podPriorityClassName: development`; under node pressure the scheduler preempts the NEWEST pod (`pod deleted` Error). Not an app/image failure — check `kubectl get events -A | grep -i preempt`. Recover: delete superseded Running workflows, resubmit the newest (`kubectl get workflow <dead> -o json` → strip to `{generateName, spec}` → `kubectl create`; the resubmit picks up the current template incl. a just-bumped pin). Durable newest-wins cancellation tracked in Forgejo `gitops#1`.
- **Rollout-window a11y findings**: a crawl overlapping a frontend rollout records `landmark-one-main`/`page-has-heading-one` on routes that rendered as the blank rollout page — verify the flagged routes serve `h1`+`main` live before treating as regressions.
- **Schemathesis `GET /Artists`-style read-timeout during a deploy** = the API restarting mid-fuzz. Cross-check the backend pod `startTime` against the workflow window.

## Schemathesis

- **Chronic `text/html` 4xx on fuzzer-mutated C1-control-char paths** (`%C2%92` etc.) is Tomcat's connector rejecting the URI before Spring's RFC7807 handler — unfixable in app code; the gate reconciles 4xx text/html as informational (5xx still blocks). Don't add these paths to `schemathesis-exclude-paths` (kills real coverage).
- **~40× "Missing Content-Type / JSON deserialization" with 429 bodies** = the fuzzer overrunning nginx `api_limit` (100 r/s) — self-inflicted. Verify the real-user 429 is conformant (parallel curl burst → `application/problem+json`); fix per the `/qa` rate-limit rule (QA-source carve-out), not app change.
- **"Missing authentication: only 401/403" has two known harness causes**: (1) invalid Origin from a base-url with a path → Spring CORS 403 `Invalid CORS request` on every request (fixed upstream, autoqa#25 — signature: all failure bodies are that string); (2) **fuzzer self-logout** — `POST /Sessions/Logout` revokes its own Bearer mid-run. `/Sessions/Logout` + `/Users/Password` are in `schemathesis-exclude-paths`; general rule: any session-terminating or credential-mutating op must be excluded from token-authenticated fuzzing.
- **Conformance levers live in `app/OpenApi*Config`, not adapter annotations** (adapters have no springdoc/swagger/jakarta-validation on the classpath by design — a swagger import there fails only in a clean CI build). Known levers: global error codes incl. 415/406, path-param patterns (`pathIdPatternCustomizer` — the biggest FP-killer), 204 via `noContentResponseCustomizer` (springdoc only infers 200), 206 on Range GETs (`partialContentResponseCustomizer`), int bounds (int32 AND int64 — every unmapped Long body field is a future red; add new DTOs to `NON_NEGATIVE_INT_BODY_FIELDS` in the same commit), body-field maxLength from DB varchar bounds, `pattern: \S` on NON_BLANK fields, enums for command fields, and **uuid `items.format` for id-ARRAY fields** (`UUID_ARRAY_ITEM_FIELDS` — a field-level `format=uuid` is a no-op on arrays).
- **`Map<String, Any>` in = `Map<String, Any>` out**: untyped map request bodies are unconstrained schemas (fuzz junk failing only at the DB); untyped map responses emit `additionalProperties` that real values violate. Type a DTO, both directions.
- **Kotlin PascalCase DTO props emit DUAL schema properties** — always camelCase props + `@JsonProperty("PascalCase")` + `@JsonAlias`.
- **Irreducible residue** (documented, left as-is): whitespace-only strings passing `minLength: 1` but failing `isBlank()` handlers; valid-format-but-nonexistent ids on create endpoints (clean problem+json 400s, reconciled upstream via autoqa#30). Do NOT chase these into per-field regex.
- **`/Lyrics/*` are external-proxy endpoints** (lrclib.net) — excluded from schemathesis (gitops sensor); covered by crawler login + integration tests. Same lever as OAuth/Logout.
- **The adaptive endpoints take snake_case body fields with NO aliases** (`seed_track_id`, `signal_type`, `expected_epoch`). A wrong-cased field parses to null → session gets no seed → queue is 100% `bootstrap-affinity` — a TEST ARTIFACT, not a bug (also bites Swift CodingKeys in the iOS app).

## ZAP

- **Boolean-SQLi HIGH on any `Principal`-parameter endpoint is a false-positive class**: unauthenticated requests 401 in the filter chain before the handler, and consistent status regardless of payload reads as injection. Also fires as 401→429 on the login path (auth rate-limiter). Fix with path PREFIXES in `zap-rate-limited-paths` (e.g. `/api/Users/AuthenticateByName`, `/api/v1/me/devices`, `/api/Sessions/Playing`) — verify via the controller source that EVERY method under the prefix takes `Principal` before widening. A SQLi HIGH on an undeclared path is a real finding.
- **A WARN storm at bare-origin paths** (`https://yay-tsa.com/v1/*`, no `/api`) means the tool scanned the wrong target, not an app/config problem — check the rewritten `servers` block in the run's `openapi-zap.json` artifact first (root-caused upstream once: autoqa#18).

### Mandatory flows (project-specific positive signals)

Health endpoints, `/Items` 200, no-pod-restart, "the page renders" are liveness only — an audio-stream 500 once shipped with all four green. Run on BOTH desktop and mobile (412×915, Pixel 7); each flow needs 0 console errors, 0 unexpected 4xx/5xx, and the named positive signal.

1. **Playback start (most critical)** — open album → Play first track. Within 3s: `<audio>` exists, `paused===false`, `currentTime>0`, `GET /api/Audio/<id>/stream` returns `206` (range) or `200`. Console must NOT contain `MEDIA_ELEMENT_ERROR` / `Failed to load audio` / `Audio engine error`.
2. **Seek** — `audio.currentTime = duration*0.5`; within 2s network shows fresh `Range: bytes=<offset>-` → 206. Repeated byte-0 request = range support broken.
3. **Next/previous** — Next fires new `/stream`, `currentTrack.Id` changes. Previous <3s in → previous track; >3s in → seeks to 0. Both required.
4. **Favorites** — heart → `POST /api/UserFavoriteItems/<id>` 200, icon fills, `/api/Items?IsFavorite=true` includes it; click again → DELETE 200.
5. **Karaoke** — toggle loads `/api/Karaoke/<id>/instrumental` 200; vocals toggle switches `/instrumental` ↔ `/vocals`.
6. **Devices / multi-device SSE** — `new EventSource('/api/v1/me/devices/events')` opens (readyState 1), receives `ready` within 5s; second tab appears within ~15s (heartbeat). SSE 500 = auth filter chain stripped the cookie (distinct from streaming 500).
7. **Radio similarity (ML actually fires)** — start a session, `POST /api/v1/sessions/{id}/queue/refresh`, then the queue's `addedReason` values must contain ≥1 `bootstrap-similarity`. Before declaring the path dead: (a) confirm the seed registered (snake_case `seed_track_id`; wrong case = seedless session = 100% affinity fill, a test artifact); (b) a seed whose own track has no embedding correctly falls back to affinity (graceful); (c) test ≥3 distinct real seeds — pick them from `track_features WHERE embedding_mert IS NOT NULL` while the extraction backlog drains.
8. **Stream sampling (highest-impact catch)** — sample ~40 random non-audiobook tracks, count `/Audio/{id}/stream` Range responses; any material 404 rate means the scanner is failing to persist tracks (see Library / scanner). Confirm: backend log `grep -cE 'idx_images_one_primary|Failed to process file'` must be 0 after a scan.

For login/seek/auth-injection mechanics, mobile locator state-changes, PWA service-worker cache, Range-aware mocks, and the binary-path rule — see `/qa` skill `Browser QA`.

### Stale service-worker cache = phantom regressions

A long-lived Playwright MCP tab (or any browser) can serve a stale SW-cached bundle across deploys — a just-removed UI element still appears, a just-shipped feature reads as absent. Before filing ANY frontend regression: (1) curl the live bundle (`curl https://yay-tsa.com/ | grep -oE 'assets/index-[^"]+\.js'`, then grep that JS for the suspect string) for server-side ground truth; (2) if server is clean, unregister SWs + delete caches in the browser and re-navigate. The loaded `/assets/index-*.js` name must match prod.

- **PWA update flow is SILENT auto-update** (no prompt): `onNeedRefresh` applies immediately when no `<audio>` is playing, else rechecks every 30s and applies at the first idle moment; `clientsClaim: true` + `skipWaiting: false`. Verify after a deploy: an idle open tab self-reloads onto the new bundle within ~15 min (or seconds after refocus); a playing tab updates after pause. Never reintroduce a manual update button without testing the claim/controllerchange chain end-to-end (the old button was silently dead — `clientsClaim: false` meant `controllerchange` never fired).
- **Stale-chunk 404s in nginx logs around a frontend rollout** are normal SPA-during-deploy behavior; `installChunkReloadRecovery()` (vite:preloadError → guarded one-time reload) handles the user side. Confirm sightings fall within a rollout window.

### Playback false-positives

- **"Spontaneous pause/skip"** — usually a legitimately short track (the library has real 12–55s interludes). Check the element's `duration` vs `RunTimeTicks`; `Seamless switch complete {seekPosition: 0}` = normal gapless advance. `document.querySelector('audio')` samples only the FIRST of the dual elements — enumerate both. For a persistent phantom pause, monkey-patch `HTMLMediaElement.prototype.pause` with a stack capture to distinguish app-initiated from browser/media-session.
- **Monkey favorite-409** under rapid heart-clicking is a legitimate OCC race. The real bug signature is an IDEMPOTENT re-favorite 409: `curl -X POST /api/UserFavoriteItems/<id>` twice sequentially — 200/200 healthy, 200/409 regression (see OCC no-op below).
- **Cloudflare QUIC + EventSource**: `net::ERR_QUIC_PROTOCOL_ERROR` console errors when a long-lived SSE stream dies (rollout, CF idle rollover) are benign — the hook reconnects. Only investigate if a fresh EventSource fails to OPEN.
- **Decorative cover 404s**: an art-less album correctly 404s `/Images/Primary`; the UI falls back to placeholder. Telemetry drops IMG resource errors at source. Don't treat as regression.

### Mobile E2E locator quirks

- Mobile viewport tests use a separate Playwright project (`[mobile]`, Pixel 7 412px); mocked specs run in `chromium-mocked` (`--project=chromium` reports "No tests found" — check `npx playwright test --list`).
- PlayerBar renders a mini-bar on mobile hidden via CSS `invisible` when the full player opens — assert `playerBar.currentTrackTitle`, not `toBeVisible()`. Test IDs must exist in BOTH mini-bar AND MobileFullPlayer for `:visible` selectors.
- `getByRole('button', { name: 'Shuffle' })` also matches the AlbumDetailPage text-button — use `locator('button[aria-label="Shuffle"][aria-pressed]:visible')`.
- Badge text may include a unit suffix ("15m") — `Number.parseInt`. `setPointerCapture` throws on synthetic pointer events — guard with try/catch. Modals must `createPortal` to escape invisible parents.
- Lighthouse LCP budget 3000ms (2500 too tight for CI). Buttons with responsive text (`hidden sm:inline`) need explicit `aria-label` + `aria-hidden="true"` on the span.
- **console-guard fixture** (`tests/e2e/fixtures/console-guard.fixture.ts`): every mocked spec imports `test`/`expect` from it. Specs with bespoke `/api` routing must call `stubSse(page)` (the app opens EventSources; a JSON catch-all logs MIME-type errors). Per-spec allowlists are **string** patterns (`test.use({ allowConsole: [...] })`) — RegExp breaks the worker boundary.
- **A mocked e2e proves UI mechanics, never endpoint support** — album/artist favorite hearts stayed green against mocked routes while prod 404'd (favorites are per-track BY DESIGN; album/artist favorite state is derived server-side, and the Subsonic adapter rejects album/artist stars). Real-endpoint specs must toggle a real track heart.
- **dnd-kit keyboard-reorder sleeps are load-bearing** — the keyboard sensor measures droppable rects async with no DOM observable; every condition-based sync flaked. The playlists S2925 findings are accepted in Sonar with this justification; new `waitForTimeout` in OTHER specs is still a finding.

## Client-error telemetry (review every pass)

The PWA ships sanitized client errors to `POST /api/v1/client-errors` (unauthenticated, bucket4j 30/min/IP); backend logs one JSON line (logger `client-error` → Loki) and bumps `yaytsa_client_errors_total{category,type,version}`. This is the ONLY window into client-only failures. **Reviewing it is the point — read the errors and fix the bugs, don't just confirm the pipe.**

- **`kubectl logs` is NOT the source of truth** — it truncates at pod start, and every backend rollout resets it AND the in-pod counter. Authoritative: **Loki** (`{namespace="yay-tsa"} |= "client_error"`, 7–30d; loki-0 has no curl and refuses cross-pod — port-forward `svc/loki 3100` in `monitoring` and query from the host) and **Prometheus** `increase(yaytsa_client_errors_total[30d])` (reconciles resets). Port-forwards are read-only, fine for `/qa`.
- **Filter before triage**: `version=unknown` = smoke traffic (`qa-smoke`) or public-endpoint probes, not app bugs; a `dropped{reason=malformed}` flood = someone probing (confirm bucket4j holds). **Filter the QA run's own traffic** — monkey/crawler beacons carry the run window's timestamps + Linux/Chrome UA; correlate before treating a fingerprint as a real user. A class appearing only on versions OLDER than its fix is fixed — confirm current version's counter is 0.
- **Triage rule (zero-deferral)**: every distinct fingerprint is a real client-only defect → reproduce + fix or `gh issue create`. Post-deploy spike on a fingerprint = regression signal, bisect by `version` tag. Redaction audit every pass: spot-check `msg`/`stack` for surviving `api_key`/`Bearer`/`token=` — a live leak means the server regex missed a vector (client redaction is bypassable).
- **Live pipeline smoke**: POST a body containing `api_key=LEAKTEST` with `Content-Type: text/plain` → expect 204 + a single redacted log line. Malformed and >16KB bodies must also 204 (never 5xx). A no-Origin POST must 204 (unload beacon has no Origin — a 403 means the origin/CSRF gate isn't exempting it). In-browser: `navigator.sendBeacon` returns true, `fetch({keepalive:true})` → 204.
- **Logging doctrine (backend-rollout burst class)**: a retry-in-progress is `debug`, not `warn`; a fire-and-forget background call uses `bestEffort` + a consecutive-failure escalator (heartbeat warns after 3 consecutive; DJ death surfaces via `MAX_REFRESH_ERRORS`). Never blanket-debug (hides outages), never blanket-warn (every rollout spams telemetry).
- **A real-user 404 on `/artists/{id}`** can be a severed bookmark (artist deleted by a later scan). Confirm the id is in NO table and the referer is the page itself; the graceful not-found branch + one beacon is correct behavior.
- **Batch-GET query-string budget**: any GET joining N ids must keep the worst-case request line well under 8KB (nginx/Tomcat header limit) — `getItemsByIds` runs 100 ids/batch for this reason; 200 produced real-user 400s.

## Backend gotchas

- **JVM heap = 75% of container limit** — raising the limit raises heap. Backend OOMKill breaks ALL active streams ("sometimes plays, sometimes doesn't"). Concurrent `/Images/Primary` decoding is the classic heap-eater (`MAX_CONCURRENT_IMAGE_PROCESSING=4` semaphore + 3Gi limit). Audio-separator needs 8Gi for BS-Roformer (OOMKilled at 6Gi under sustained batch; `_release_separation_memory()` gc's after every separation).
- **Health grep before declaring a release healthy**: `kubectl logs deploy/<backend> --since=5m | grep -E "Unhandled|MessageNotWritable|MessageConversion"`. `HttpMessageNotWritableException` = "5xx with body the client can't decode". Async `AsyncRequestNotUsableException` on Pause/Skip is benign (swallowed no-body handler). Client-abort IOExceptions on the binary stream path (`Broken pipe`/`Connection reset`/`Host is unreachable` during flush) are swallowed to debug by `isClientAbort` — `grep -A3 "Unhandled exception" | grep -iE "Broken pipe|Connection reset"` must be 0.
- **One-thread TaskScheduler starvation**: any long @Scheduled work run inline freezes the 1s outbox poller — SSE commands/events pile up unpublished while HTTP stays green. Diagnose: `SELECT context, published_at IS NULL, count(*) FROM core_v2_shared.outbox GROUP BY 1,2` — unpublished rows on a healthy pod = starved scheduler. Long scheduled work must dispatch to its own executor (karaoke separation does; `spring.task.scheduling.pool.size: 4` as defense).
- **OCC no-op incompatibility**: a domain no-op returning Success with an UNCHANGED version breaks an OCC repo that derives `expectedVersion = newVersion - 1` → zero rows matched → 409 on every idempotent re-favorite / OpenSubsonic re-star. Persist only when `newVersion != snapshot.version`. Black-box detect: double-POST an already-favorite id (see Browser QA).
- **Composite-key first-write races need native upserts**: Spring Data `merge()` (SELECT-then-INSERT) on a composite `@IdClass` double-INSERTs under concurrent first play → `resume_position_pkey` violations. Native `INSERT … ON CONFLICT DO UPDATE` + `@Modifying(clearAutomatically = true, flushAutomatically = true)` (clearAutomatically REQUIRED or `@DataJpaTest` re-reads stale managed entities). Detect: `grep -c resume_position_pkey` = 0.
- **Delete-then-insert on a shared unique index needs an explicit flush between** — Hibernate orders inserts before deletes in a flush batch. See Library / scanner for the production instance.
- **Missing multipart part → 500 unless mapped**: `MissingServletRequestPartException` is NOT `MultipartException`; map it → 400 alongside `MissingServletRequestParameterException`. Multipart parsing runs BEFORE auth — a non-admin token reaches the 500.
- **jackson-module-kotlin null-in-collection**: JSON `null` inside a non-nullable `List<String>` NPEs in the controller → 500 without `KotlinFeature.StrictNullChecks` + `HttpMessageConversionException` → 400 mapping (write-side `HttpMessageNotWritableException` stays 500). NB: `NewStrictNullChecks` needs jackson-module-kotlin 2.19+; a warm local daemon can "compile" the wrong feature name — only clean CI is authoritative.
- **NUL bytes**: NUL in a query-string VALUE passes StrictHttpFirewall (path-only) and PostgreSQL rejects it (`SQLState 22021`) with WARN+ERROR log spam. Defense: `LibraryQueries.scrubQuery` strips NUL at the search entry, and a global Jackson NUL-stripping String deserializer scrubs every write field at the edge. Detect after any fuzz window: `grep -c 22021` = 0 (measure AFTER the fuzz, not on a fresh pod — a clean grep on a just-restarted pod proves nothing). **Tooling trap**: writing a `" "` escape via the Edit tool plants a literal 0x00 byte in source (even in comments — file goes `Binary` in git); construct NUL as `0.toChar()` and check `git diff` for `Binary files differ` before committing anything that mentions NUL.
- **Raw DB-exception messages must not reach RFC7807 `detail`** — `DataIntegrityViolationException.message` carries the full INSERT with schema/columns/values. Genericize DB-level violations (log server-side); domain InvariantViolations keep their message. Detect: trip any constraint and grep the response `detail` for `insert into`/`core_v2_`.
- **Best-effort background ops log ALL typed failures at debug** — reflection (`ReflectExternalPlayback`) retries only Conflict/StorageConflict; every other typed Failure (unknown track from a fuzz UUID, lost lease) is a business outcome, not operator-actionable. When downgrading a best-effort op's logging, cover ALL its Failure variants — a fuzzer surfaces the ones prod doesn't.
- **BOLA write check**: every endpoint accepting a `UserId`-ish request param must 403 on mismatch with principal (`grep -n "UserId(userId" adapter-*/**/*Controller.kt` — createPlaylist/getPlaylistItems are the reference pattern). The read-side fix once shipped while the write-side param slipped.
- **Poison signal rows**: an absurd `positionMs` in `playback_signals.context` overflows the int4 sum every aggregator tick — the whole affinity pipeline stops while HTTP stays green. Clamp at ingestion AND in the aggregate SQL AND in stats reads.
- **Personalized-branch dead-fallback distinguisher**: the personalized `/Items` branch reports its OWN page size as `TotalRecordCount`; the fallback reports the full library count. `curl '…&SortBy=DatePlayed&Limit=5&ExcludeGenres=audiobook,audiobooks'` → `TotalRecordCount=5` = branch fired; full count = dead fallback (a guard on `excludedGenres.isEmpty()` once killed it for every real user, since the PWA always sends ExcludeGenres).
- **Remote-command targeting**: commands target the TOKEN-BOUND deviceId; the PWA's local `yaytsa_device_id` can drift on legacy sessions and the fail-closed check silently drops every command. The heartbeat echoes the registered identity and the client adopts it (`getEffectiveDeviceId()`). When "remote command has no effect": compare the SSE event's `targetDeviceId` against the browser's effective id.
- **MCP**: tools are E2E-testable via plain `POST /api/mcp` (JSON-RPC `tools/call`) with the QA Bearer — no claude.ai connector, no admin. `playback_sessions` populated via `ReflectExternalPlayback` (change-gated) — play in the PWA, `GET /api/Sessions` shows `NowPlayingItem` within ~10s; a session leased by another device rejects reflection by design. claude.ai caches `tools/list` at connect time — new tools require a reconnect, not a bug. Cloudflare blocks `Python-urllib` UA on POSTs (403 before origin) — scripted QA must use a browser-ish UA. `queue_entries` id columns are varchar(36) — longer ids fail only at flush.

## Library / covers / scanner

- **Cross-schema track-id refs are by-value, no FK, in 11 tables** — any dedup/cleanup must remap all of them before dropping an `entities` row.
- **Schema features ship dead** — ML embeddings/HNSW/affinity existed in the DB for months with no production reader. Treat every new schema feature as untrusted until an integration test exercises a controller/worker through it. Operational signal: Micrometer `yaytsa.ml.affinity.last_updated_at` + Grafana >6h-stale alert.
- **Test-vs-prod column-type drift**: Hibernate ddl-auto in Testcontainers creates jsonb where prod (Flyway/ETL) has text — jsonb `->>` passes tests, floods prod with `BadSqlGrammarException`. Portable form: `substring(col::text from '<regex>')`. Grep prod for `BadSqlGrammar` after any worker-SQL change. Prod runs `ddl-auto: validate`.
- **Placeholder vs missing tags**: `##### ######` in TITLE/ARTIST (Cyrillic encode failure) → `usableTag()` treats `^[#?\s]+$` as missing → filename fallback (strips leading track numbers). A tag that IS a filename leaks the audio extension into the title — `stripTrailingAudioExtension` handles scan-time; detect leftovers: `SELECT count(*) … WHERE entity_type='TRACK' AND name ~* '\.(flac|mp3|wav|ogg|m4a|aac|wma|opus)$'` = 0. The M1–M4 mechanical walkthrough checks catch this class.
- **Scanner delete+insert cover trap (highest-impact class)**: replacing a Primary image via `delete(existing); save(new)` in one tx violates `idx_images_one_primary` (Hibernate flushes inserts before deletes), the deferred failure surfaces from an unrelated native query (stack blames `linkGenre` — red herring; read the failing SQL), and the whole `upsertTrack` tx aborts → the track keeps a stale ETL path → clean 404 problem+json on stream for a large library fraction while `/Items` and `/health` stay green. Fix pattern: explicit `flush()` between delete and save. Detect via the ~40-track stream sampling flow (Browser QA #8).
- **Image read-path**: `/Items/{id}/Images/Primary` = image row → embedded art (own file, or album's first track) → 404. `images.entity_id` joins `entities.id` (NOT `entities.entity_id`) — the wrong join reads as "no images". Audiobook chapter tracks carry no own image row and the track DTO emits no `AlbumPrimaryImageTag` — covers come by AlbumId with a synthetic tag fallback client-side. Check `ffprobe … attached_pic` before treating an `/Images/Primary` 404 as a data bug (audiobook rips often carry art only in tags).
- **Cover acquisition** (`MetadataEnricher`): music albums → CAA (by MB release-group) → iTunes → Deezer; audiobook albums are EXCLUDED from music APIs (same-named music album's art) — dedicated per-track pass (borrow-parent → CAA-by-mbid → Open Library). All external clients throw `MetadataProviderUnavailableException` on 429/5xx/timeout so the entity is parked+retried, NEVER burned: a client returning empty on transient failure permanently skips the row via `metadata_checked_at` (a whole outage cycle silently burns every row it touches; re-queue burned rows by clearing `metadata_checked_at` on rows with no MBID — on the primary). The audiobook path must ALSO park on the exception path with a shorter provider cooldown, or the failing head starves the OFFSET-0 candidate tail (symptom: dozens of `cover deferred` WARNs/hour all citing the SAME release-group while coverless-count doesn't move).
- **Artist/cover caches live on the media volume via subPath** (`.yaytsa/artist-images`, `.yaytsa/cover-cache`) — emptyDir caches broke artist images on every rollout while DB rows persisted. The candidate query treats any existing row as "has image" even if its file is gone — legacy ephemeral-path rows must be cleaned once so the backfill re-fetches. The cover lands under `music-path`, so the media mount must be writable in prod (`backend.media.readOnly: false`) — read-only silently skips art while MBIDs still set.
- **Offline cover pre-cache must NOT gate on a `Primary` tag** (backend often omits it on track items even when the album has art). A failed cover cache is best-effort debug, never error telemetry.
- **Egress smoke from the pod**: busybox `wget` to musicbrainz returns **403** (blank UA blocked) — that 403 still proves DNS+connectivity; the real client sends the mandatory UA. CAA 307-redirects to archive.org (the part that usually times out).
- **TotalRecordCount must be a DB count, not `items.size`** — infinite scroll stops at `loaded >= totalRecordCount`. Sanity: `Limit=1` response's TotalRecordCount == DB count.
- **Audiobook exclusion is layered**: (1) main library via `ExcludeGenres` param; (2) adaptive surfaces via the backend `MusicSurfaceFilter` (RadioSeed DTO carries no genre — client can't filter); (3) client belt-and-suspenders in DailyMix. **When excluding a track class from recommendations, patch `MusicSurfaceFilter` (the common exit for daily-mix/discovery/personalized/radio-seeds), not an individual candidate source** — similarity legs resurface what a random-fill-only fix misses. Subsonic `getRandomSongs` is deliberately unfiltered. Live check: pull the personalized feed with `Fields=Genres` 3× (random fill resamples), assert 0 audiobook-genre items. `Genres=Audiobook` as a query param is NOT the mechanism (returns the full set). DB baseline: audiobook tracks are ~2.4% of the library, so an unfiltered feed leaks ~2 per 95-item page statistically.
- **CD-filler silence tracks are real library content** ("Silent"/"Blank" 9–10s padding rips): karaoke separation legitimately per-file-fails them (`Audio file … is empty or not valid` — SHOULD burn fail_count), and recommendation surfaces drop them via `MIN_RANDOM_TRACK_DURATION_MS` + `MusicSurfaceFilter`. Distinguish from the separator-outage regression by the `Separator returned 500` message shape.

## Workers (karaoke / ML / enricher / LLM-DJ)

- **Stems**: `/app/stems/{trackId}/{filename}_instrumental.wav` + `_vocals.wav`; the stems PVC must be mounted in BOTH separator (writes) and backend (reads) — else `validateStemFilesExist()` resets `karaokeReady=false`. `tryRecoverFromOrphanedFiles()` rebuilds DB state from existing stems. `karaoke_fail_count` parks a track after 3 failures.
- **Infrastructure outage must never consume a per-item retry budget** (doctrine, two instances): a separator connection-class `IOException` (`ConnectException`/null message/`header parser received no bytes`) maps to `SeparatorUnavailableException` — no fail_count, batch aborts until next tick; an HTTP-status failure still burns budget (genuine per-file). Detect: `SELECT fail_count, last_error, count(*) FROM core_v2_karaoke.assets WHERE ready_at IS NULL GROUP BY 1,2` — `fail_count>=3` rows with connection-class `last_error` are a regression; the asset row is a regenerable projection (absence = unprocessed → re-enters batch). Same doctrine as the enricher park rule.
- **On-demand karaoke priority**: `POST /Karaoke/{trackId}/process` queues into a `ConcurrentLinkedQueue` drained BETWEEN batch tracks — a user request waits one separation, not the whole backlog. `processTrack` is idempotent. The port is injected nullable (bean exists only when karaoke enabled). Verify: toggle an unprocessed track → PROCESSING → READY in ~minutes even with a batch running.
- **Feature-extractor starvation + parking**: permanently-failing files (WMA, >30min FLACs) at the head of `ORDER BY created_at` eat the whole nightly batch — `core_v2_ml.extraction_failures` + max-attempts anti-join parks them. Check every pass: `track_features` count grows ~batchLimit/night until the backlog drains; the failures table explains what parked and why. essentia's TF wrapper spams 30k WARN lines/run unless silenced (`essentia.log.warningActive = False`) — that spam once rotated the real failure lines out of `kubectl logs`.
- **CronJob image pins are manual** (Image Updater can't track CronJobs — no live pod): featureExtractor/embeddingServer/lyricsSync/tasteClusters share one `services/audio-ml` image hand-pinned in gitops `values.images.yaml`. **Check every pass: pin sha vs `git log main -1 -- services/audio-ml`.** Cheap staleness detector: `processed=N failed=19` night after night with `extraction_failures` EMPTY = the running image predates the parking fix. A broken shared module (`COPY` list missing `db.py`) crashes every CronJob at import while playback stays green — check for CronJob pods in `Error`, and verify a fixed image with a one-off `kubectl create job --from=cronjob/<lightest>` (delete after).
- **ArgoCD app health can wedge `Degraded`** after a CronJob pod failure even once everything is healthy — stale computed field, clears on the next real resource transition. Trust live signals; do NOT restart the argocd controller (admin namespace, forbidden).
- **Affinity reconciliation (current formula)**: `AffinityAggregator` folds BOTH `playback_signals` AND non-adaptive `play_history` on separate watermarks. Healthy: `SUM(play_count)` < PLAY_START signals + non-adaptive history plays (audiobooks excluded from the fold); `SUM(skip_count)` = SKIP signals + history `skipped=true`. **The double-count trap inflates ABOVE total events** — that's the red flag. Doctrine: a watermark/cursor migration onto a pre-populated additive projection must seed the cursor to the prior watermark OR truncate+replay; never seed epoch against a non-empty additive table. (The old signals-only equality is stale — don't use it.)
- **LLM-DJ**: `LlmClient` calls the in-cluster LiteLLM gateway (`http://litellm.litellm.svc.cluster.local:8080`, OpenAI-compatible, model `GPT-5.4 Mini`), not the Anthropic API. Go-live proof: `grep LlmOrchestrator` shows `LLM-DJ added N tracks`. Wiring (all gitops): litellm's `gatewayClientNamespaces` must include `yay-tsa`; `LLM_API_KEY` (dedicated virtual key, SOPS) in the prod secrets; `llm.enabled: true`. The key has a hard $/30d budget — exhaustion degrades to ML-only gracefully.
- **LLM-DJ over-spend check**: a correct run is near-silent — `grep -c 'LLM-DJ added'` over 2min ≈ sessions with NEW signals (single digits). A steady ~15/min = the freshness gate regressed (must skip a session whose newest signal id equals the last decision's `triggerSignalId`); fix the gate, never disable the DJ. Token-audit columns are unpopulated — count log lines or read the key's spend.
- **Chart's own NetworkPolicy defaults to OFF** (`backend.networkPolicy.enabled: false` in the OSS chart, for portability); production must set it `true` explicitly — the central gitops network-policies chart covers DB/traefik/monitoring but knows nothing about LiteLLM egress, so with the flag off the default-deny baseline blocks the backend's own egress to litellm with an immediate **"Connection refused"** and the DJ silently degrades forever. Diagnose from inside the pod (`kubectl exec … wget litellm…/health/readiness`); "Connection refused" despite litellm's correct ingress policy = the EGRESS side. ## Routing / edge / CSP

- **Multi-protocol path catalog**: PWA → `/Users/AuthenticateByName`, `/Sessions/*`, `/Items*`, `/Audio/{id}/stream`, `/Playlists*`, `/UserFavoriteItems/*`, extensions `/v1/*` (test via `/api/` prefix — bare `/v1/*` hits frontend nginx and returns index.html). Subsonic at `/rest/*` (class-level mapping, NOT `/api/rest`). MPD in-cluster only (ClusterIP :6600, unauthenticated, no ingress; verify via backend startup log — `/dev/tcp` probe fails on JRE-Alpine busybox). `PlaylistItemId` on the wire is the playlist POSITION (protocol clients use it; the PWA never does).
- **The real edge is the frontend nginx, not Traefik ingress priorities**: the frontend `PathPrefix(/)` ingress wins on rule-length priority over explicit low priorities, so `/api` and `/rest` work only because nginx proxies them (`location /api/`, `location /rest/` — no `/api` strip on /rest, keep Range for `stream.view`). schemathesis tests only `/api` — it never catches a broken `/rest`. Verify Subsonic: `curl '…/rest/ping.view?u=test&p=<pw>&f=json'` → subsonic-response JSON, not HTML. Isolate routing-vs-adapter by port-forwarding the backend service directly.
- **Consequence for headers/CSP**: `/api/*` flows through the frontend router and picks up the SHARED kube-system CSP — a CSP fix on the v2 `secure-headers` middleware has ZERO effect unless a dedicated ingress with `router.priority: "100"` + strip + secure-headers wins for that specific path (`ingress-oauth.yaml` does this for `/api/oauth`). Only paths that genuinely need the v2 CSP get the high-priority ingress — everything else keeps nginx's strip-prefix/X-Accel behavior. Diagnose by curling the Traefik ClusterIP with a Host header (bypasses Cloudflare) and checking which CSP comes back.
- **Cloudflare injects a default CSP when the origin sets none** (incl. `form-action 'self'`), and passes an origin-set CSP through unchanged. Chrome enforces `form-action` against the WHOLE redirect chain — the MCP OAuth consent 302 to `https://claude.ai/...` (or loopback for Claude Code) is blocked by a bare `form-action 'self'`, and the symptom reads as "new MCP tools not appearing" (the user can't reconnect). The CSP must list the client origins at the ORIGIN level (Traefik middleware; backend `OAuthHtmlPages` emits the registered-redirect-origin form-action as defense-in-depth). Verify: `curl -sI …/api/oauth/authorize… | grep -o "form-action[^;]*"`.
- **`.well-known/oauth-*` routing**: only the frontend-nginx proxy location works (the chart's dedicated wellknown ingress never matches — same catch-all class). When OAuth discovery 404s, check WHICH server answered (nginx HTML vs problem+json) before touching the backend. OAuth endpoints are deliberately hidden from OpenAPI (fuzzing unauthenticated DCR pollutes `oauth_clients`).
- **HSTS**: server-level `add_header` does NOT reach proxied locations that include their own header set — HSTS must be at server level with `always`, or ZAP flags every `/api/v1/*` 401.
- **Auth surfaces**: only `Authorization: Bearer` survives the Cloudflare/Traefik chain; `X-Emby-Token`/`?api_key=` 401 through the public URL but work via port-forward — code that authenticates via port-forward but fails publicly = proxy-chain bug. The PWA writes a `yay_token` cookie for `<audio src>` and EventSource (can't set headers).

## Tests (Gradle / Testcontainers)

- **`:app:test` shared-context flake lottery — known classes** (one Spring context, one Postgres, committed seeds, no rollback; a DIFFERENT test fails each run, masking that the causes are systemic):
  1. **Auth rate-limit bucket sharing** — MockMvc defaults to one `remoteAddr`; token-issuing logins must `.with(uniqueClientIp())`.
  2. **Scheduled library scan reconciling the empty test music-path** — deletes just-seeded entities mid-`@BeforeEach` (FK violations). Gated behind `yaytsa.scanner.scheduled-enabled=false` in the test base; don't "fix" by pointing music-path elsewhere.
  3. **Capped "full listing" assertions** — seeds from other classes push yours past `MAX_PAGE_SIZE`; page until exhausted and assert membership over the union.
  4. **Hardcoded values in UNIQUE columns** — `@BeforeEach` re-runs per method; key every seeded UNIQUE column on a per-invocation UUID. The failing test's body is a red herring — read the test-report XML artifact for the real constraint name.
  5. **Query-count guards under full-suite state** (suspected, tracked #284) — SQL-log the class when it re-fires.
- **Testcontainers reused PG**: parallel module JVMs share one Postgres; `CREATE EXTENSION IF NOT EXISTS` is not concurrency-safe (`pg_extension_name_index` duplicate) — extension creation is wrapped in an advisory lock + pre-created for all modules. A single green re-run does NOT disprove a race.
- **Run the changed module's WHOLE `:test` task before pushing a domain fix** — an existing test may encode the old buggy behavior; decide bug-encoding (update assertion, with comment) vs regression (revert), never blindly flip green.
- **Gradle warm-daemon/cache traps**: a green `./gradlew build` right after creating a NEW file can skip compiling it; `:app:cleanTest :app:test` can be served FROM-CACHE — a rerun proves nothing unless `--no-build-cache`/`--rerun-tasks` and the task line says `executed`. Only clean CI is authoritative.
- **detekt** is live build-wide on all 26 modules; three load-bearing pieces: the kotlin-compiler-embeddable pin MUST be registered inside `afterEvaluate` (BOM downgrade re-breaks it otherwise); PascalCase naming allowance for Jellyfin wire DTOs (don't "fix"); frozen per-module baselines — regenerate ONLY to shrink, never to absorb new findings. Baselines key on signatures: a type-only change to a grandfathered function invalidates its entry (1-line `detektBaseline` regen, not a refactor).
- **ktlint one-item-per-line for multi-line collection literals**: a compact hand-edit can pass a stale cached `spotlessApply` and fail CI's clean `spotlessKotlinCheck`. Run `./gradlew spotlessCheck` (whole project, cheap) as the last step before every backend commit.

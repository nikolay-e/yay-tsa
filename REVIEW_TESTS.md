## Round 2 — Synthesis (systemic patterns)

1. **Infra/worker modules ship with NO test source set.** Of 18 Gradle modules, only `app` and `core-testkit` have `src/test`. Zero in `infra-llm/-ml-worker/-library-scanner/-media/-notifications/-karaoke-worker`, all `infra-persistence/*` except where noted, every `adapter-*`. Root cause: convention = "test through `app` `@SpringBootTest`," but workers run outside the HTTP context → fall through the only gate. Fix: a Testcontainers harness per infra-worker module.
2. **Core guarantees proven only in simulation.** `SpringTransactionalCommandExecutor.kt` is referenced by **no test**; every OCC/outbox/lease/idempotency test uses `DirectTransactionalExecutor`/`OccSimulatingExecutor`+`InMemory*`. Fix: re-run the core-testkit scenario suite bound to the real executor + Testcontainers (parameterized).
3. **Schema/feature exists but no behavior test reaches it** — `findSimilarTracks`, `findSimilarByMert/Clap/Discogs`, `filterRedLines`, `buildRecommendedSeeds`, `OutboxPoller` appear in no test source. Same structure as the 6-month ML dead-code failure. Fix: Konsist/ArchUnit rule requiring each `*QueryPort` similarity method + poller to have a reachable integration test.
4. **Adapters tested in isolation; one-core-many-protocols invariant never asserted.** Fix: write-via-A/read-via-B suite + parameterize the testkit registry across all four protocols.
5. **Shipped-bug fixes lack regression guards** — ~15+ QA.md fixes, each unguarded. Fix: a standing "QA-regression" suite, one test per QA.md entry.

- **Meta root cause:** a single test chokepoint (`app` `@SpringBootTest`) with no Postgres-backed harness for the components that hold the system's guarantees.

## Final Verdict

### TL;DR

yay-tsa's test gaps are not random — they cluster precisely where the post-deploy autoqa pipeline is blind: **data-dependent correctness** (is this recommendation personalized? is this scanned title clean? is this codec's Content-Type right?), which a well-formed HTTP `200` silently satisfies. The six background-worker / streaming modules that hold these behaviors ship with **no test source set at all**, and ~15 documented production fixes carry no regression guard — so the same bug classes recur. The fixes are cheap; the project just has no test surface positioned to catch them.

### Systemic Patterns

1. **Six worker/edge modules have no test source set** — verified: only `app`, `core-testkit`, and every `infra-persistence/*` submodule have `src/test`; `infra-ml-worker`, `infra-library-scanner`, `infra-media`, `infra-karaoke-worker`, `infra-llm`, `infra-notifications` have **none**. They run outside the `app @SpringBootTest` chokepoint — the only test gate — so they fall through entirely. Root cause + highest-leverage fix: a Testcontainers harness per worker module (reuse the shared-postgres container).
2. **Autoqa-blind correctness** — the post-deploy pipeline asserts shape/status/render, never _semantic correctness of data_. Every surviving 🔴 is a behavior where a wrong answer is still a valid `200`.
3. **Shipped fixes without regression guards** — ~15 QA.md entries, each hand-patched in prod, none with a test. Fix: a standing "QA-regression" suite, one test per QA.md bug class.
4. **Guarantees proven only in simulation** — `SpringTransactionalCommandExecutor` is referenced by no test; OCC/outbox/lease are exercised via `DirectTransactionalExecutor`/`InMemory*` only (though JPA-layer OCC _is_ covered at `PlaybackSessionRepositoryTest.kt:107`).
5. **One protocol tested, four shipped** — `TestCapabilities.kt:107` `testCapabilitiesRegistry()` is Jellyfin-only; MPD writes silently `ok()` (`MpdCommandHandler.kt:58-64`); cross-protocol state consistency never asserted.

### False Positives / Overstated (from R1, corrected in R2/R3)

- "Only `app`+`core-testkit` have tests" — **wrong**: all nine `infra-persistence/*` submodules have `src/test`. The real gap is the 6 worker/edge modules.
- Outbox exactly-once / crash-redelivery (R1 🔴) — downgraded: the SSE consumer (`useDeviceEvents`) is idempotent (absolute last-write-wins), so duplicate delivery is cosmetic.
- Two-device lease contention (R1 🔴) — downgraded: unreachable for a solo user; real-PG OCC already proven at `PlaybackSessionRepositoryTest.kt:107`.
